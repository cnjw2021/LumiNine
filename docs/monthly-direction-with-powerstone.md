## 第1部: 월단위(月盤) 길방위 로직 설계

### 1. 설계 개요

기존 연단위 흉방위 판별 로직(9가지 조건)은 **판별 엔진 자체는 동일**하게 재사용하며,  
입력값만 **연반(年盤) → 월반(月盤)**, **年破 → 月破**로 교체하는 구조이다.

```
[연단위]                          [월단위]
연반성 배치(年盤) ──┐             월반성 배치(月盤) ──┐
年干支 ─────────────┤             月干支 ─────────────┤
본명성(本命星) ─────┼→ 흉방위     본명성(本命星) ─────┼→ 흉방위
월명성(月命星) ─────┤  판별엔진   월명성(月命星) ─────┤  판별엔진
상성매트릭스 ───────┘  (공통)     상성매트릭스 ───────┘  (공통)
```

따라서 월단위 확장에 필요한 **신규 로직**은 다음 2가지이다:

1. **월반성(月盤星) 배치 산출 로직**
2. **월간지(月干支) 산출 로직**

---

### 2. 월반성(月盤星) 배치 산출 로직

#### 2-1. 월의 시작 기준: 절입일(節入日)

구성기학에서 "월"은 양력 1일이 아니라 **24절기의 절입(節入)**을 기준으로 바뀐다.

| 월 | 절기(節氣) | 대략적 양력 날짜 |
|----|-----------|----------------|
| 1월(寅月) | 立春 (입춘) | 2/3~2/5 |
| 2월(卯月) | 啓蟄 (경칩) | 3/5~3/7 |
| 3월(辰月) | 清明 (청명) | 4/4~4/6 |
| 4월(巳月) | 立夏 (입하) | 5/5~5/7 |
| 5월(午月) | 芒種 (망종) | 6/5~6/7 |
| 6월(未月) | 小暑 (소서) | 7/6~7/8 |
| 7월(申月) | 立秋 (입추) | 8/7~8/9 |
| 8월(酉月) | 白露 (백로) | 9/7~9/9 |
| 9월(戌月) | 寒露 (한로) | 10/8~10/9 |
| 10월(亥月) | 立冬 (입동) | 11/7~11/8 |
| 11월(子月) | 大雪 (대설) | 12/6~12/8 |
| 12월(丑月) | 小寒 (소한) | 1/5~1/7 |

**구현 포인트**:
- 절입일은 매년 미세하게 달라지므로, **천문력 데이터(에페메리스)** 또는 **신뢰할 수 있는 절기 DB**를 참조해야 한다.
- Python에서는 `ephem` 라이브러리 또는 국립천문대(NAOJ) 공개 데이터를 활용 가능.
- こよみすと 유파에서는 절입 시각까지 고려하므로, 날짜뿐 아니라 **시각 단위**의 절입 데이터가 필요하다.

#### 2-2. 월반 중궁성(月盤中宮星) 산출

연반 중궁성으로부터 월반 중궁성을 결정하는 규칙이다.

**Step 1: 해당 연도의 연반 중궁성 그룹 확인**

연반 중궁성은 3그룹으로 분류된다:

| 그룹 | 연반 중궁성 | 별칭 |
|------|-----------|------|
| 上元(상원) | 1, 4, 7 | 一白・四緑・七赤 |
| 中元(중원) | 2, 5, 8 | 二黒・五黄・八白 |
| 下元(하원) | 3, 6, 9 | 三碧・六白・九紫 |

**Step 2: 그룹별 1월(寅月, 立春~) 월반 중궁성 결정**

| 연반 중궁성 그룹 | 1월(寅月) 월반 중궁성 |
|-----------------|---------------------|
| 上元 (1, 4, 7)  | **8** (八白土星)      |
| 中元 (2, 5, 8)  | **5** (五黄土星)      |
| 下元 (3, 6, 9)  | **2** (二黒土星)      |

**Step 3: 월이 진행될 때마다 중궁성이 역행(逆行)한다**

월반성은 **역순(감소 방향)**으로 순행한다:  
`... → 9 → 8 → 7 → 6 → 5 → 4 → 3 → 2 → 1 → 9 → 8 → ...`

예시) 연반 중궁성이 4(上元 그룹)인 해:

| 월(節月) | 월반 중궁성 |
|---------|-----------|
| 1월(寅) | 8 |
| 2월(卯) | 7 |
| 3월(辰) | 6 |
| 4월(巳) | 5 |
| 5월(午) | 4 |
| 6월(未) | 3 |
| 7월(申) | 2 |
| 8월(酉) | 1 |
| 9월(戌) | 9 |
| 10월(亥) | 8 |
| 11월(子) | 7 |
| 12월(丑) | 6 |

**산출 공식**:

```python
def get_monthly_center_star(year_center_star: int, month_index: int) -> int:
    """
    월반 중궁성을 산출한다.
    
    Args:
        year_center_star: 해당 연도의 연반 중궁성 (1~9)
        month_index: 절월 기준 월 (1=寅月/立春~ ... 12=丑月/小寒~)
    
    Returns:
        월반 중궁성 (1~9)
    """
    # Step 1: 연반 중궁성 그룹별 1월(寅月) 시작값
    if year_center_star in (1, 4, 7):
        start = 8  # 上元
    elif year_center_star in (2, 5, 8):
        start = 5  # 中元
    elif year_center_star in (3, 6, 9):
        start = 2  # 下元
    
    # Step 2: 역행 (월이 진행될수록 감소)
    result = start - (month_index - 1)
    
    # Step 3: 1~9 범위 순환
    result = ((result - 1) % 9) + 1
    
    return result
```

#### 2-3. 월반성 8방위 배치

중궁성이 결정되면, 9개 별을 8방위 + 중궁에 배치하는 로직은 **연반과 동일한 낙서(洛書) 비행순서**를 따른다.

```
낙서(洛書) 기본 배치:      비행 순서(궁 순서):
┌────┬────┬────┐
│ SE │  S │ SW │          中→NW→W→NE→S→N→SW→E→SE
│  4 │  9 │  2 │          (중궁에서 시작하여 이 순서로 배치)
├────┼────┼────┤
│  E │ 中 │  W │
│  3 │  5 │  7 │
├────┼────┼────┤
│ NE │  N │ NW │
│  8 │  1 │  6 │
└────┴────┴────┘
```

**비행 순서 인덱스**: `[中, NW, W, NE, S, N, SW, E, SE]`

중궁에 배치될 별 번호가 결정되면, 그 별부터 시작하여 순차적으로(순행: +1씩 증가) 위 궁 순서에 배치한다.

```python
FLIGHT_ORDER = ['center', 'NW', 'W', 'NE', 'S', 'N', 'SW', 'E', 'SE']

def place_stars_on_board(center_star: int) -> dict:
    """
    중궁성을 기준으로 9개 별을 8방위 + 중궁에 배치한다.
    
    Args:
        center_star: 중궁에 배치될 별 (1~9)
    
    Returns:
        {방위: 별번호} 딕셔너리
    """
    board = {}
    for i, position in enumerate(FLIGHT_ORDER):
        star = ((center_star - 1 + i) % 9) + 1
        board[position] = star
    return board
```

**검증 예시**: 중궁 = 5일 때 (기본 낙서 배치)

| 궁 순서 | 방위 | 별 |
|---------|------|-----|
| 0 | 中 | 5 |
| 1 | NW | 6 |
| 2 | W | 7 |
| 3 | NE | 8 |
| 4 | S | 9 |
| 5 | N | 1 |
| 6 | SW | 2 |
| 7 | E | 3 |
| 8 | SE | 4 |

→ 기본 낙서와 일치하므로 정상.

### 3. 월간지(月干支) 산출 로직

月破를 판정하기 위해 해당 월의 간지(干支)가 필요하다.

#### 3-1. 월지(月支) - 고정 대응

절월과 지지(地支)는 고정 대응이다:

| 절월 | 지지(地支) |
|------|----------|
| 1월 | 寅 |
| 2월 | 卯 |
| 3월 | 辰 |
| 4월 | 巳 |
| 5월 | 午 |
| 6월 | 未 |
| 7월 | 申 |
| 8월 | 酉 |
| 9월 | 戌 |
| 10월 | 亥 |
| 11월 | 子 |
| 12월 | 丑 |

#### 3-2. 월간(月干) - 年干에 의해 결정

연간(年干)에 따라 1월(寅月)의 천간이 결정되고, 이후 순행한다.

**年干 → 1월(寅月) 천간 대응표** (五虎遁 / 고호둔):

| 年干 | 1월(寅月)의 天干 |
|------|----------------|
| 甲, 己 | 丙 |
| 乙, 庚 | 戊 |
| 丙, 辛 | 庚 |
| 丁, 壬 | 壬 |
| 戊, 癸 | 甲 |

이후 월이 진행될 때마다 천간이 순행(+1):  
`甲→乙→丙→丁→戊→己→庚→辛→壬→癸→甲→...`

```python
CELESTIAL_STEMS = ['甲','乙','丙','丁','戊','己','庚','辛','壬','癸']
TERRESTRIAL_BRANCHES = ['寅','卯','辰','巳','午','未','申','酉','戌','亥','子','丑']

# 五虎遁: 年干 → 1월(寅月)의 天干 인덱스
GOKOTON_MAP = {
    0: 2,  # 甲 → 丙
    5: 2,  # 己 → 丙
    1: 4,  # 乙 → 戊
    6: 4,  # 庚 → 戊
    2: 6,  # 丙 → 庚
    7: 6,  # 辛 → 庚
    3: 8,  # 丁 → 壬
    8: 8,  # 壬 → 壬
    4: 0,  # 戊 → 甲
    9: 0,  # 癸 → 甲
}

def get_monthly_kanshi(year_stem_index: int, month_index: int) -> tuple:
    """
    월간지(月干支)를 산출한다.
    
    Args:
        year_stem_index: 해당 연도의 천간 인덱스 (0=甲 ~ 9=癸)
        month_index: 절월 기준 월 (1=寅月 ~ 12=丑月)
    
    Returns:
        (천간, 지지) 튜플
    """
    # 1월(寅月)의 천간 인덱스
    first_month_stem = GOKOTON_MAP[year_stem_index]
    
    # 월 진행에 따라 순행
    stem_index = (first_month_stem + (month_index - 1)) % 10
    branch_index = month_index - 1  # 0=寅, 1=卯, ...
    
    return (CELESTIAL_STEMS[stem_index], TERRESTRIAL_BRANCHES[branch_index])
```

#### 3-3. 月破 산출

月破는 해당 월의 지지(地支)의 **정반대 방위**이다.  
기존 연단위 年破 로직과 동일한 대충(対冲) 테이블을 사용한다:

| 지지 | 반대 지지 | 반대 방위 |
|------|----------|----------|
| 子 | 午 | S (남) |
| 丑 | 未 | SW (남서) |
| 寅 | 申 | SW (남서) |
| 卯 | 酉 | W (서) |
| 辰 | 戌 | NW (북서) |
| 巳 | 亥 | NW (북서) |
| 午 | 子 | N (북) |
| 未 | 丑 | NE (북동) |
| 申 | 寅 | NE (북동) |
| 酉 | 卯 | E (동) |
| 戌 | 辰 | SE (남동) |
| 亥 | 巳 | SE (남동) |

### 4. 흉방위 판별 엔진 (공통 / 기존 로직 재사용)

연단위와 월단위에서 공통으로 사용하는 판별 엔진의 입력-출력 인터페이스:

```
[입력]
├── board: dict           # 구성반 배치 {방위: 별번호}  ← 연반 or 월반
├── main_star: int        # 사용자 본명성 (1~9)
├── month_star: int       # 사용자 월명성 (1~9)
├── break_direction: str  # 破 방위 (年破 or 月破)
└── compatibility_matrix  # 상성 매트릭스

[출력]
└── directions: dict
    └── {방위: {
            is_auspicious: bool,
            marks: list[str]   # 해당 흉살 마크 목록
        }}
```

**9가지 판별 조건 (연단위와 100% 동일)**:
1. 五黄殺: board[방위] == 5
2. 暗剣殺: 오황살의 정반대 방위 (중궁 5일 때 없음)
3. 本命殺: board[방위] == main_star
4. 月命殺: board[방위] == month_star
5. 水火殺: (board[S]==1) or (board[N]==9) 일 때 해당 1 또는 9의 방위
6. 破: 방위 == break_direction
7. 凶方星: compatibility_matrix[main_star][board[방위]] == BAD
8. 本命的殺: 본명살 방위의 정반대
9. 月命的殺: 월명살 방위의 정반대

### 5. 멱등성(冪等性) 보장 설계

동일 입력에 대해 항상 동일 출력을 보장하기 위한 설계 원칙:

| 항목 | 보장 방법 |
|------|----------|
| 절입일 판정 | 외부 천문력 API가 아닌 **사전 계산된 절입일 DB**를 내장. 최소 100년분(1925~2025+)을 정적 데이터로 보유 |
| 월반 중궁성 | 순수 산술 연산 (나눗셈/나머지). 외부 의존 없음 |
| 월간지 | 순수 산술 연산. 외부 의존 없음 |
| 흉방위 판별 | 결정론적 조건문. 랜덤 요소 없음 |
| 상성 매트릭스 | 정적 테이블. 변경 불가 상수 |

**핵심**: 절입일 데이터만 정적으로 확보하면, 나머지는 모두 결정론적 산술이므로 멱등성이 자동 보장된다.

## 第2部: 파워스톤 매칭 설계

### 1. 매칭 철학

구성기학과 파워스톤은 모두 **오행(五行)** 사상을 공유한다.  
이 공통 기반을 활용하여 **3레이어 추천 체계**를 설계한다.

```
[Layer 1] 基本石 (기본석)     ← 본명성(本命星) 기반 / 평생 불변
[Layer 2] 月運石 (월운석)     ← 그 달의 길방위 오행 기반 / 매월 변동
[Layer 3] 護身石 (호신석)     ← 그 달의 흉방위 대비 기반 / 매월 변동
```

---

#### 2-1. 구성(九星) → 오행 대응

| 구성 | 한자 | 오행 |
|------|------|------|
| 1 | 一白水星 | 水 |
| 2 | 二黒土星 | 土 |
| 3 | 三碧木星 | 木 |
| 4 | 四緑木星 | 木 |
| 5 | 五黄土星 | 土 |
| 6 | 六白金星 | 金 |
| 7 | 七赤金星 | 金 |
| 8 | 八白土星 | 土 |
| 9 | 九紫火星 | 火 |

#### 2-2. 방위 → 오행 대응

| 方位 | 오행 |
|------|------|
| N (北) | 水 |
| NE (北東) | 土 |
| E (東) | 木 |
| SE (南東) | 木 |
| S (南) | 火 |

| W (西) | 金 |

#### 2-3. 오행 → 파워스톤 대응표

각 오행에 대응하는 대표 파워스톤을 **주석(Primary)**과 **부석(Secondary)**으로 분류한다.

| 오행 | 색상 계열 | 주석 (Primary) | 부석 (Secondary) |
|------|----------|---------------|-----------------|
| **水** | 청색·흑색·투명 | 아쿠아마린 (Aquamarine) | 라피스라즐리 (Lapis Lazuli), 블루토파즈 (Blue Topaz), 오닉스 (Onyx) |
| **木** | 녹색·청록 | 에메랄드 (Emerald) | 페리도트 (Peridot), 아벤츄린 (Aventurine), 제이드 (Jade) |
| **火** | 적색·자색·주황 | 가넷 (Garnet) | 카넬리안 (Carnelian), 루비 (Ruby), 자수정 (Amethyst) |
| **土** | 황색·갈색·베이지 | 시트린 (Citrine) | 타이거아이 (Tiger's Eye), 옐로재스퍼 (Yellow Jasper), 스모키쿼츠 (Smoky Quartz) |
| **金** | 백색·금색·은색 | 수정 (Clear Quartz) | 문스톤 (Moonstone), 로즈쿼츠 (Rose Quartz), 진주 (Pearl) |

### 3. Layer 1: 基本石 (기본석) - 본명성 기반

사용자의 본명성에 따라 **평생 고정**되는 기본 파워스톤이다.

| 본명성 | 오행 | 기본석 | 의미 |
|--------|------|--------|------|
| 一白水星 | 水 | 아쿠아마린 | 유연함과 지혜의 흐름 |
| 二黒土星 | 土 | 시트린 | 대지의 포용력과 안정 |
| 三碧木星 | 木 | 에메랄드 | 성장과 새로운 시작의 에너지 |
| 四緑木星 | 木 | 페리도트 | 조화와 인연을 잇는 힘 |
| 五黄土星 | 土 | 타이거아이 | 중심을 잡는 강인한 의지 |
| 六白金星 | 金 | 수정 (Clear Quartz) | 순수한 판단력과 리더십 |
| 七赤金星 | 金 | 로즈쿼츠 | 사교성과 매력을 높이는 기운 |

**설계 근거**:
- 같은 오행(예: 木)이라도 구성의 성격 차이를 반영하여 주석/부석을 차별 배정한다.
- 三碧(활동적 木) → 에메랄드(강한 성장), 四緑(온화한 木) → 페리도트(조화)

### 4. Layer 2: 月運石 (월운석) - 월별 길방위 기반

그 달의 **길방위 중 가장 에너지가 강한 방위**의 오행에 대응하는 스톤을 추천한다.

#### 4-1. 최적 길방위 선택 로직

```
[월반 흉방위 판별 결과]
    ↓
[모든 길방위(is_auspicious = True) 추출]
    ↓
[우선순위로 정렬]
    1순위: 본명성과 상성이 GOOD인 방위
    2순위: 본명성과 상성이 NEUTRAL인 방위
    ↓
[동순위 시 방위 고정 우선순위 적용]
    S > E > SE > SW > N > W > NE > NW
    ↓
[최적 길방위 확정]
    ↓
[해당 방위의 오행 → 파워스톤 매칭]
```

#### 4-2. 매칭 규칙

1. 최적 길방위의 오행을 확인한다.
2. 해당 오행의 **주석(Primary)**을 월운석으로 추천한다.
3. 만약 주석이 Layer 1의 기본석과 중복되면, **부석(Secondary)**의 첫 번째를 추천한다.

#### 4-3. 예시

> 본명성: 三碧木星, 해당 월의 최적 길방위: 南(S)  
> → 남의 오행 = 火  
> → 火의 주석 = 가넷  
> → 기본석(에메랄드)과 미중복 → **가넷** 추천

### 5. Layer 3: 護身石 (호신석) - 흉방위 대비

그 달의 **가장 강력한 흉살**이 걸린 방위의 오행을 **극(剋)하거나 설(洩)하는** 오행의 스톤을 추천한다.

#### 5-1. 흉살 위험도 순위

```
1위: 五黄殺 (가장 강력한 대흉살)
2위: 暗剣殺
3위: 本命殺
4위: 月命殺
5위: 水火殺
6위: 破
7위: 凶方星
8위: 本命的殺
9위: 月命的殺
```

#### 5-2. 오행 상극(相剋) 관계를 이용한 호신석 결정

흉살이 걸린 방위의 오행이 사용자에게 나쁜 기운을 보내므로,  
그 나쁜 기운을 **억제(剋)** 하거나 **흡수(洩)** 하는 오행의 스톤을 배치한다.

| 흉방위 오행 | 억제하는 오행 (相剋) | 호신석 오행 |
|-----------|-------------------|-----------|
| 水 | 土가 水를 막음 | **土** |
| 木 | 金이 木을 자름 | **金** |
| 火 | 水가 火를 끔 | **水** |
| 土 | 木이 土를 뚫음 | **木** |
| 金 | 火가 金을 녹임 | **火** |

```
[가장 위험한 흉살이 위치한 방위]
    ↓
[해당 방위의 오행 확인]
    ↓
[상극 관계로 억제하는 오행 결정]
    ↓
[해당 억제 오행의 주석을 호신석으로 추천]
    ↓
[기본석·월운석과 중복 시 부석으로 대체]
```

#### 5-3. 예시

> 해당 월 가장 강력한 흉살: 五黄殺이 西(W)에 위치  
> → 서의 오행 = 金  
> → 金을 억제하는 오행 = 火  
> → 火의 주석 = 가넷  
> → 만약 가넷이 이미 사용 중이면 → 카넬리안 추천

### 7. 파워스톤 매칭 시 중복 회피 규칙

3개의 레이어에서 같은 스톤이 추천되는 것을 방지한다:

```
1. 기본석 확정 (본명성 → 고정)
2. 월운석 후보 결정 → 기본석과 같으면 부석[0]으로 대체
3. 호신석 후보 결정 → 기본석·월운석과 같으면 부석[0] → 부석[1]로 순차 대체
```

이 규칙에 의해 항상 **3개의 서로 다른 스톤**이 추천된다.

```
         ↑                   │          ↑                   │
         └───────────────────┘          └───────────────────┘
         
        木生火: 나무가 불을 키움        木剋土: 나무가 흙을 뚫음
        火生土: 불이 재(흙)를 만듦      土剋水: 흙이 물을 막음
        土生金: 흙에서 금속이 남        水剋火: 물이 불을 끔
        金生水: 금속 표면에 물이 맺힘   火剋金: 불이 금속을 녹임
        水生木: 물이 나무를 키움        金剋木: 금속이 나무를 자름
```
